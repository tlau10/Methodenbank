2. Solution of the model of linear programming

2.1 Graphical solution

As already indicated in chapter 1, a graphical solution is possible for the two-dimensional case (2 variables). For the solution procedure we need several steps:

  • depiction of the restriction as straight lines in the coordinate system.
  • ascertain the solution space.
  • drawing the iso profit straight line for any Z-element.
  • parallel shifting of the isoprofit straight line so that there is at least one point of intersection with the solution space, and it is as far away as possible from the origin.
  • this point of intersection with the solution space is the optimal solution.

In the case of the updated example given above, the point of intersection of the straight lines I and II results in the coordinates:

     x1 = 18/7;   x2 = 15/7

The profit contribution is:

     Z = 18/7 * 1 + 15/7 * 2 = 48/7 = 6 6/7  [€]

In practise, such graphical solution methods are not relevant, as there are only two dimensions we can depict. Therefore, we strive for a mathematical method. The basis for this shall be indicated in the following paragraph.

2.2 The Simplex principle

In the example given, the optimal solution point resulted in point C, that is in the point of intersection between the straight lines I and II. Other objective functions result in different solution points:

example 1:

If we cannot obtain a profit contribution with product 1, the effects on the objective function are as follows:

ZF: 2x2 = Z

This leads to the solution:

x2 = 3 ;  x1 = 0 ;  Z = 6.

With this, we have found the solution point B.

example 2:

Let the objective function be as follows:

ZF: 1 * x1 + 3 * x2 = Z

Solution:
every point of the line between the two points BC is a solution, as the direction of the straight line of the objective function coincides with the direction of the line between the two points BC. Among others, the following points are solutions:

x1 = 0 ;  x2 = 3  und  Z = 9 [€]

x1 = 18/7 ;  x2 = 15/7

Z = 18/7 + 45/7 = 63/7 = 9 [€]

We can see from these examples that the optimal solution point is always a corner of the convex solution polyhedron. This phenomenon provides the reason for the precondition to only allow convex solution polyhedrons for such settings of a problem. We call this precondition the Simplex principle:

In looking for the optimal solution point, it is sufficient to analyse the corners of the convex polyhedron that encloses the solution space M. With this, we can realise a first mathematical solution method: the complete enumeration.


2.3 Solution through enumeration

The solution of a problem of linear programming can now be solved with the following algorithm:

Step 1:
Calculate all corners of the polyhedron that encloses the solution space M (in the example, the coordinates for A, B, C, D). This is equivalent to calculating the points of intersection of straight lines.

Step 2:
Insert the calculated corners into the objective function. The corners with the highest value of the objective junction is the solution.

In the given example, we now want to carry out the calculation of the points of intersection of the straight lines.
The precondition of mathematically ascertaining the points of intersection is a linear equation system. Since the restrictions are marked with a ""-relation, we will transform these with the aid of a special technique into "="-relations.

restrictions:

    3x1 + 2x2 12
     x1 + 3x2  9

Now we add a further two variables, x3 and x4, which take on the difference of left and right side in the case of non-equality. Therefore, these variables are also called slack variables:

    3x1 + 2x2 + x3 = 12
     x1 + 3x2 + x4 =  9

The variables x1 and x2 are called structure variables, the variables x3 and x4 (are called) slack variables.

The slack variables can be interpreted economically: they can be understood as unused capacities:

    x3 indicates the zero capacity of machine 1, and
    x4 indicates the zero capacity of machine 2.

Example:  with x1 = 2,5 and x2 = 2 , both the slack variables x3 and x4 result in the value 0.5.

The modification we have carried out on the model of linear programming has now 4 variables and 2 equations. For the solution space, this results in an expected ambiguous solution:

The variables x1 and x2 are called independent variables, because the calculation of their value is dependent to several operations. By contrast, the variables x3 and x4 can in this case be regarded as the function of the variables x1 and x2, as well as of the right and left side (b1 and b2, respectively). x3 and x4 are therefore called dependent variables.

In order to be able to ascertain the points of intersection of the straight lines, we let the independent values arbitrarily be zero; we can then immediately see the value of the dependent variables. In the case of the above matrix, this results in x3 = 12, x4 = 9, and x1 = 0, x2 = 0. In the two-dimensional depiction in fig.2, this point of intersection would be A, the origin of the coordinate system. The point of intersection of two straight lines is also called the basic solution. The points of intersection of individual restriction lines (including the coordinate axes, which also enclose the solution space) then represent the set of all basic solutions.

The bounded variables are called basic variables, the unbounded variables non-basic variables. A basic solution then consists of the basic variables (in this case x3 = 12, x4 = 9) and the non-basic variables (x1 = 0, x2 = 0). We also recognise the basic variables in the matrix representation, where they have columns with only one single 1 and otherwise only zeros. Of course, the individual basic variables have the 1 in different rows.

We get all the points of intersection of the restrictions together, and with the coordinate axes, by ascertaining all the basic solutions of the system. The number of basic variables is always equal to the number of restrictions. Because of this, with four variables and two equations, we get six pair combinations altogether. In developing a more efficient solution method, we will thus try to make use of the information about the model in a way that we do not even get nonpermitted basic solutions. The following improvements of the method of enumeration are thus a kind of acceleration function (tuning). Such an accelerated method, which is still based on the Gauss method, is called Simplex method.


2.4 The Simplex method

2.4.1 Calculating the permitted basic solution

For calculating only the permitted basic solutions, we expand the Gauss method by adding a further criterion for selecting the Pivot row. This criterion avoids selecting a Pivot element that results in an nonpermitted basic solution. If we go back again to the initial matrix:

    3
    1     		2
    3     		1
    0     		0
    1     		|
    |     		12
    9

If we want x1 to be in the basis, both rows could become Pivot rows. If the first row is the Pivot row, then x1 and x4 are the basic variables, x2 and x3 are the non-basic variables. In this case, the previous basic variable x3 disappears in favour of the new basic variable x1. We call this fact a basis change. That in this case the variable x1 takes on the value 12/3 = 4 is easy to see. For the second row, we get - since the non-basic variables x2 and x3 take on the value zero - the equation:

Inserting x1 = 4 yields: 

   4 + x4 = 9  x4 = 5

With this basis solution, we would have ascertained corner D again.

On the other hand, we could have selected the second row as the Pivot row. The second equation yields a basis change of the variable x4 in favour of the variable x1. The new basic variable x1 takes on the value.

   9/1 = 9 an.  (x1 = b2/a21)

Per definition, the non-basic variables x2 and x4 both take on the value zero. Inserting the ascertained value in the first equation yields: 

   3 x1 + x3 = 12
     27 + x3 = 12

From this yields:

      x3 = -15

In this case, the result is an nonpermitted basic solution. Interpreting this economically, we can say that with full working to capacity of the first machine, the second machine would be used beyond capacity, and therefore its slack variable x3 takes on a negative value. In this case, the first restriction proves to be "narrower" than the second restriction. The criterion we need to include in order to avoid nonpermitted basic solutions now becomes a search for the bottleneck capacity. In this case, we need to select that row as Pivot row that leads to the lower value of x1. Only then can we avoid negative values. We can formalise this observation in the Pivot criterion: The selection criterion for the Pivot row reads:


    ;   ais > 0



Inserting this criterion for the Pivot row in the enumeration method, we can indeed avoid getting an nonpermitted basic solution. In the initial matrix, a decision for the first column as Pivot column yields to a decision for the first row as Pivot row, because the latter's quotient 12/3 = 4 is less than the one of the second row: 9/1 = 9.

With this, the top left element of the matrix becomes the Pivot element and leads to the basic solution of corner D. In analogy, by selecting the second column as Pivot column we must select the second row as Pivot row:

    9 : 3 = 3 is less than 12 : 2 = 6

Hence we get the basic solution of point B, and not the nonpermitted basic solution x2 = 6, x4 = -9.

Selecting a Pivot element is only for those rows whose coefficients of the Pivot column are greater than zero. Otherwise, the variable xj would in this case also be < 0. We have now defined an algorithm that leads exclusively to permitted basic solutions. However, there is a further step of acceleration for this example.

Out of the infinite solution alternatives of the solution polyhedron M, we were able to achieve a limitation down to 6 solution points through enumeration. With the help of the Pivot number criterion, this number was reduced to 4 permitted basic solutions. With the optimality test, which will be discussed in the following chapter, we can achieve a further reduction to 3 basic solutions for the given example.

2.4.2 Calculating the optimal basic solution

In our calculations so far, we have not considered the information that follows out of the depiction of the objective function. Making use of this, we can achieve a further reduction of the basic solutions we need to ascertain. To do this, we expand our equation system by including the objective function. Z is a variable whose solution can be ascertained as part of the Gauss algorithm:

   3 x1 + 2 x2 + x3      = 12
   1 x1 + 3 x2      + x4 =  9
   1 x1 + 2 x2           =  Z

Bringing the variable Z to the left side:

   3 x1 + 2 x2 + x3         = 12
   1 x1 + 3 x2      + x4    =  9
   1 x1 + 2 x2           -Z =  0

Since the variable Z is always meant to be calculated in its positive value, we need to make it the basic variable. For this, the positive coefficient is essential:

   3 x1 + 2 x2 + x3          = 12
   1 x1 + 3 x2      + x4     =  9
  -1 x1 - 2 x2           + Z =  0

In the initial matrix, the value of the objective function (bz) is zero. As the objective of the problem is maximisation, we need to ask how we can increase the value of bz. For this we need to analyse how the value of bz comes about in the individual matrix transformations. For this we carry out a matrix transformation using the Gauss algorithm; we arbitrary select the first column as the Pivot column.

Because of the Pivot column criterion, we can only select the top left element a11 as Pivot element. The first row needs to be divided by 3. This modified first row must now be subtracted from the second row, and added to the third row, respectively, in order to be able to allocate the value zero to the remaining coefficients of the first column. Generally, we would say that the modified first row is multiplied by a21, and subtracted from the second row. The same applies for the third row; here we only multiply by c1. As can be seen in table 3, the value of b2 in the resulting matrix is:

   bz(n+1) = bz(n) - c1' * b1/a11    (c1' = -c1)

Generally applies for the value of bz in the resulting matrix:

   bz(n+1) = bz(n) - br(n)/ars(n) * cs'(n)
   
      n  iteration index
      r  Pivot row
      s  Pivot column

We can see here that bz(n+1) only becomes greater if cs' is less than zero.

With this we have now deduced the optimality test of the Simplex method: the value of the objective function can only be increased if a coefficient of the objective function cj' (n) 0 exists in the matrix representation. If cj' (n) 0 applies, then we have found the optimal solution.

Thus, if we have a matrix whose coefficients of the objective function have an exclusively positive value, we can deduce that the ascertained basic solution must be optimal. On the other hand, if we have negative coefficients of the objective function, the selection of the Pivot column is limited to those columns that have negative coefficients of the objective function.
If there are several negative coefficients of the objective function, the selection of the Pivot column is done arbitrarily: we select the smallest cj' (n)-value as Pivot column. In many cases, this is a quicker way to get to the optimal solution; however, we cannot mathematically prove this statement. Thus, the selection criterion for the Pivot column is an arbitrary procedure. It reads:

    min j Î { column indices }       { cj' (n) }

      Method called "steepest ascent unit" !

Of course, we consider only those column indices whose coefficient of the objective function has a minus sign.

With this, we have indicated all the possibilities to accelerate the originally employed enumeration method. We now want to revise the individual steps of the Simplex method.


2.4.3 The steps of the Simplex method

The Simplex method is an iterative procedure, i.e. we carry out several steps with the same operation. For the representation of the matrices, there is a special writing convention:

1st step: inserting formulation in initial tableau
2nd step: finding out whether negative coefficients of the objective function exist
yes: carry on with 3rd step.
no:   stop. optimal solution found
3rd step: select Pivot column j
min j Î { column indices }     { cj' (n) }
4th step: select Pivot row i:
min i Î { row indices }     { bi' (n) / ais(n) };     ais(n) > 0
5th step: carry out row transformations according to the Gauss algorithm;
continue with 2nd step
