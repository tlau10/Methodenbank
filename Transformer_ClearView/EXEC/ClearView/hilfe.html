ClearView Hilfedatei

Systemvoraussetzungen

ClearView läuft auf jedem Windows PC, der die folgenden Bedingungen erfüllt:

  1. Eine Installation vom Visual Studio .Net,
  2. Eine Version des Internet Explorers muss auf dem System,
  3. Das Verzeichnis c:\temp muss vorhanden sein und der Lesen und Schreiben muss darauf erlaubt sein.
  4. Java muss in der Version 1.4.2_03 installiert sein und als aktuelle Standard Java Anwendung definiert worden sein.

Installation

ClearView wird durch das Kopieren der nötigen Dateien in ein beliebiges Verzeichnis auf der lokalen Festplatte installiert. Es werden keine Einträge in die Systemregistrierung vorgenommen.

 

Die folgenden Dateien werden für den Einsatz von ClearView benötigt

  1. ClearView.exe : die eigentliche Anwendung
  2. Transformer.jar
  3. jakarta-regexp-1.3.jar
  4. dom4j-full.jar
  5. LP-Schema.xsd : Das XML Schema zur Prüfung der XML Dateien
  6. genhtml.xslt: Das XSLT Stylesheet zur Erstellung der HTML Statistiken
  7. hilfe.html : die Hilfedatei
  8. CVConfig.xml: die Konfigurationsdatei, in der die Pfade zur java.exe und transformer.jar gespeichert werden.

 

Bedienung

Die Oberfläche von ClearView besteht aus drei Anzeigeseiten und vier Menüs.

Beim Anzeigen von den XML Daten ist zu beachten, dass nur die Daten angezeigt werden können, welche der Transformer aus der Ausgabedatei des Solvers erstellen konnte.

Anzeigeseite Einfach

Auf dieser Seite erscheint in der Liste die Variablen und deren Ergebnisse.

Im Feld „Zielfunktion“ erscheint das Ergebnis der Zielfunktion.

Im Feld „Solver“ erscheint der Name des verwendeten Solvers und des Ausgabeformats

Das Feld „Nachricht“ zeigt eventuelle Nachrichten des Solvers an, wie z.B. wenn das Ergebnis nicht gültig ist.

 

Anzeigeseite System

Auf dieser Seite wird das Ausgangssystem dargestellt.

Die Spalten sind:

Name

 Hier steht der Name der Restriktion, und die Zielfunktion wird als „ZF“ gekennzeichnet.

„Min“ und „Max“ bezeichnen die Ober- und Untergrenzen der Variablen. Diese werden nur angezeigt, falls diese definiert wurden.

Namen der Variablen

Nach den Namen der Funktion folgen die Bezeichnungen der Variablen. In dieser Spalte stehen bei den Restriktionen den jeweiligen Faktor der Variable. Ist das Feld leer, so gilt in der Regel, dass der Wert implizit 0 dort ist. Sollte jedoch auch der Operator und der B Wert fehlen, so wurde die Restriktionen deklariert, aber nicht definiert. Dies kommt hauptsächlich bei Ergebnissen vom LP-Solve vor.

Operator

 Bei den Restriktionen wird dort angezeigt, wie der Operator dort ist. Gültige Werte sind: <, <=, =, >, >=. Bei der Zielfunktion wird hier fest „->“ angezeigt.

B

 In dieser Spalte werden bei der Zielfunktion die Optimierungsrichtung, also „Min!“ oder „Max!“ angezeigt. Bei den Restriktionen erscheint dort der jeweilige B Wert.

Als Zeilen erscheinen

  1. Die Zielfunktion
  2. Eventuell vorhandene Restriktionen
  3. Eventuell vorhandene Unter- und Obergrenzen.

Anzeigeseite Details

Auf dieser Seite werden alle relevanten Daten dargestellt.

Die ersten Spalten sind wie bei der Anzeigeseite System aufgebaut, mit dem Unterschied, dass bei der Zielfunktion in der Spalte Operator nun die Optimierungsrichtung angezeigt wird und in der B Spalte der Zielfunktionswert. Dazu kommen noch weiter Spalten:

Dualwert

Hier werden die Werte der Dualwerten zu der jeweiligen Restriktion angezeigt.

Slack

Sollte die Restriktion definiert worden sein, so wird hier der Wert des Slacks angezeigt. Sollte der Wert negativ oder Operator der Restriktion =, > oder >= sein, so ist der Wert immer 0.

Außerdem wird eine weitere Zeile hinzugefügt, welche die Reduced Costs der Variablen ausgibt.

Die letzte Zeile „Variablenwerte“ nennt nochmal die Ergebnisse der jeweiligen Variablen.

Menü Datei

Öffnen

Dieser Befehl öffnet einen Dialog, in dem sie eine XML Datei mit LP Daten öffnen können. Sobald die Datei ausgewählt und bestätigt wurde, wird intern das Dokument mit dem vorhandenen Schema geprüft. Stimmt die Datei nicht mit den dort definierten Regeln überein, so wird eine Fehlermeldung ausgegeben.

Die in der XML Datei vorhandenen Daten werden dann in den drei Anzeigeseiten dargestellt.

Speichern

Hier kann eine vorhandene XML Datei unter einem anderen Namen gesichert werden, oder eine neu erstellte Datei gesichert werden. Wenn eine Datei angezeigt wird, kann der neue Name und der Speicherort im Dialog ausgewählt werden. Dann wird die Datei dorthin kopiert.

Einstellungen...

In dem sich öffnenden Dialog können die Pfade zu der benötigten Java Anwendung und zum Transformer angesehen und geändert werden. Mit der Schaltfläche „OK“ werden die Einstellungen übernommen und der Dialog geschlossen, mit „Abbrechen“ wird nur der Dialog beendet.

Schließen

Die aktuelle Datei wird geschlossen.

Export nach HTML

Dies hat die selbe Wirkung wie das gleichnamige Menü.

Menü Bearbeiten

Dieses Menü hat einen einzigen Punkt:

System aus Zwischenablage kopieren

Das Erstellen eines Systems über die Zwischenablage erfolgt so:

  1. Eine Solver Ausgabedatei wird in einem beliebigen Editor geöffnet.
  2. Der gesamte Inhalt wird ausgewählt. Modernere Editoren erlauben hier den Tastaturkürzel „STRG+A“.
  3. Der ausgewählte Text wird kopiert. Normalerweise reicht hier der Tastaturkürzel „STRG+C“
  4. ClearView wird aufgerufen und der Menüpunkt ausgewählt. Alternativ kann der Tastaturkürzel „STRG+V“ aufgerufen werden.

Nun wird im Verzeichnis c:\temp die Datei clearview.xml erstellt, die alle Daten aus dem Zwischenspeicher enthält. Das Ergebnis sollte jedoch abgespeichert werden, da diese Datei bei jeder Operation neu angelegt wird.

Sollte es zu einem Fehler kommen, so erscheint eine Fehlermeldung.

 

Menü Export nach HTML

Beim Anklicken des Menüs oder der Schaltfläche wird ein Dialog geöffnet, in dem der Name und er Pfad der zu neu erstellenden HTML Datei mit den LP Daten eingetragen werden können. Nach dem Bestätigen wird die HTML Datei erstellt und, falls der Internet Explorer auf dem System installiert ist, wird dieser geöffnet und die Datei angezeigt.

 

Menü ?

Info…

In einer Dialogbox stehen die Entwickler, die Version und das Copyright.

Hilfe…

Mit dem Internet Explorer wird die Datei hilfe.html aufgerufen.

 

Fehler

Es gibt Umstände, in denen ClearView nicht das macht, was zu erwarten wäre. Folgende Fehlerfälle wurden beobachtet:

Beim Einlesen aus der Zwischenablage erscheint die Fehlermeldung „Temporäre Datei kann nicht gefunden werden“

In diesem Fall war der Inhalt in der Zwischenablage kein lineares Programm, welches der transformer verarbeiten konnte. Dadurch erzeugte dieser nicht die Datei Clearview.xml, die nun aufgerufen wird.

Beim Anzeigen der Hilfe erscheint eine Fehlermeldung

Die Datei Hilfe.htm“ muss sich im Verzeichnis mit der Clearview.exe befinden. Außerdem muss der Internet Explorer auf dem Rechner zur Verfügung stehen.

Beim Erstellen der HTML Statistik erscheint eine Fehlermeldung

Die Datei genhtml.xslt muss sich im Verzeichnis mit der Clearview.exe befinden.

Beim Öffnen einer XML Datei erscheint eine Fehlermeldung

Beim Öffnen einer vorhandenen XML Datei wird diese mit dem Schema aus  LP-Schema.xsd  verglichen, die sich  Verzeichnis mit der Clearview.exe befinden muss.

Befindet sich in der Fehlermeldung der Hinweis „Validationsfehler“, dann wird danach erklärt, warum diese XML Datei nicht gültig ist. Öffnen Sie eine andere Datei oder korrigieren Sie den Fehler in der Datei.

Bei der Ausgabe sind einige Felder leer

Die Qualität der Ausgabe ist nur so gut wie die Daten, die in der XML Datei stehen. Diese Daten wiederum stammen von verschiedenen Solvern, welche unterschiedlich viele Daten zur Verfügung stehen. Leere Felder können in der Regel als „0“ interpretiert werden. Da es jedoch Ausnahmefälle gibt, wird auf eine Vorbelegung mit „0“ verzichtet, um keine Fehler anzuzeigen.
