Bekannte Fehler und mögliche Verbesserungen

Verbesserungsvorschläge (zu V2.0):

·Fehlerkontrolle (speziell Einlesen der Ergebnisdatei eines Solvers)

Implementierung weiterer Solver (da bereits der Aufruf von den Solvern für das MPS-Format vorhanden ist, jedoch gibt es keinen Standart für die Ergebnisdateien der jeweiligen Solver)

·Nochmalige Überprüfung des Dateiaufbaus (bei einer Implementierung der n-dimensionalen Verschnittproblematik)

·Einbau einer Routine, die bei Fehlen einer VERSCHNITT.INI diese mit Default-Werten sofort erstellt.

·Der Dateiaufbau der VERSCHNITT.INI ist mangelhaft. Sinnvoll wäre eine neue Struktur derer. So könnten statisch gehaltene Daten leichter dort abgespeichert werden. Auch wäre die Zusammenfassung aller INI-Dateien in eine OR.INI sinnvoll. So könnte darin stehen:

·Aktivierte Solver (Kürzel kommt von hier, nicht von Datei)

·Programmpfade für Solver

·Eingabe-, Steuer- und Ausgabedateien für Solver

·Namen PIF - Dateien den jeweiligen Solveraufruf

·Bevorzugter Solver (nicht vom Programm vorgegeben)

·die vier zuletzt benutzten Modelle

·Beispiel für eine VERSCHNITT.INI

[Solver]

XaMps = True

XaEqs = True

Mops =True

Omp = False

[Solvertext]

XaMps = XA-Solver MPS-Style

XaEqs = XA-Solver Equation-Style

Mops = Mops-Solver

[Path]

XaEqs = C:\OR\SOLVER\XA

XaMps = C:\OR\SOLVER\XA

Mops = C:\OR\SOLVER\MOPS\CLIENT

[PIF-Files]

XaEqs = XA.PIF

XaMps = XA.PIF

[Verschnitt]

Extension = VRS

File1 = C:\OR\ OR-MBANK\VERSCHN\DATEN\BSP01.VRS

File1 = C:\OR\ OR-MBANK\VERSCHN\DATEN\HUGO.VRS

File3 =

·Im Hauptfenster unter Menüpunkt Datei sollten die letzten 4 benutzten Dateien aufgeführt werden und damit zugänglich gemacht werden. (Siehe Dateiaufbau für VERSCHNITT.INI)

·Im Hauptfenster, Menüpunkt Optionen, Unterpunkt Verzeichnisse taucht für jeden Solver ein Auswahlfenster für den Datenpfad auf. Sinnvoll wäre ein eigener Unterpunkt für den Datenpfad, sowie für jeden einzelnen Solver.

·Bisher wurde die Anzahl der verfügbaren Ausgangswerkstücke vernachlässigt. Bei der jetzigen Lösung des jeweiligen Verschnittproblems wird die Anzahl der benötigten Werkstücke ermittelt als Summe der Schnitte der jeweils gewählten Varianten. Es gibt aber auch die Problematik, daß nur beschränkt Ausgangswerkstoffe da sind. Im Vorlesungsskript findet sich auf Seite 38 Aufgabe 6 ein Beispiel dafür. Sinnvoll wäre also eine Erweiterung, die neben Anzahl eines Ausgangswerkstoffes auch die Möglichkeit für mehrere Ausgangswerkstoffe eröffnet.
