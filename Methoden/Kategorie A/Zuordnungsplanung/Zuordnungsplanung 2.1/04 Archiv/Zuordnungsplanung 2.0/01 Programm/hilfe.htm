Im Personalbereich stößt man immer wieder auf das Zuordnungsproblem, wenn bestimmte Stellen mit den dafür am besten geeigneten Bewerbern zu besetzten sind. Mit diesem Tool zur Zuordnungsplanung können solche Probleme komfortabel gelöst werden.

Die in Java implementierte Software verwendet neben den Solvern LP und MOPS auch einen selbst entwickelten Enumerationsalgorithmus um die optimale Lösung zu jedem Zuordnungsproblem zu finden. Die Lösungszeiten werden bei der Lösung immer mit ausgegeben. So kann auch die Performance der unterschiedlichen Solver bzw. Algorithmen ermittelt werden. Außerdem ist das Programm so gestaltet, dass auch beliebige andere Zuordnungsprobleme tabellarisch beschrieben und gelöst werden können.

Die Beispiele in der folgenden Hilfe beziehen sind jedoch immer auf ein Zuordnungsproblem im Personalbereich.

Die Oberfläche des Tools ist bewusst einfach gehalten und im wesentlichen selbsterklärend.
Trotzdem sind im folgenden alle Bedienungsmöglichkeiten, Dialoge und Menüs ausführlich erklärt.

Für die Ausgabe der Ergebnisse wird ein neues Lösungsfenster geöffnet.

 Hauptmenüleiste

1. Hauptmenüleiste: Datei

1.1 Neu

Es wird eine 3*3 Matrix für ein Zuordnungsproblem im Personalbereich angelegt.

1.2 Öffnen...

Es kann eine vorhandene Datei vom Typ *.csv geöffnet werden. Besitzt diese kein gültiges *.csv Format, so wird eine Fehlermeldung ausgegeben.

1.3 Speichern

Das aktuelle Zuordungsproblem wird im *.csv Format gespeichert.

1.4 Speichern unter...

Das aktuelle Zuordnungsproblem wird nach Selektion des entsprechenden Ordners im *.csv Format gespeichert

1.5 Beenden

Das Programm wird beendet, nachdem abgefragt wurde, ob das aktuelle Problem bereits bespeichert ist. 


2. Hauptmenüleiste: Solver

2.1 Solver LP

Zur Lösung des Zuordnungsproblems wird nach Selektion dieser Dialogoption der LP Solver verwendet.

2.2 Solver MOPS

Zur Lösung des Zuordnungsproblems wird nach Selektion dieser Dialogoption der MOPS Solver verwendet.

2.3 Solver Enumeration

Zur Lösung des Zuordnungsproblems wird nach Selektion dieser Dialogoption der von den Autoren der Software entwickelte Enumerationsalgorithmus verwendet.

3. Hauptmenüleiste: ?

3.1 Hilfe

Das Hilfe-Fenster wird geöffnet.

3.2 Info

Ein Fenster mit Hinweisen zu der Programmversion und den Autoren wird geöffnet.



 Symbolleiste und Eingabematrix

1. Button: Neu

2. Button: Öffnen

3. Button: Speichern

4. Button: Solver LP wählen

5. Button: Solver MOPS wählen

6. Button: Solver Enumeration wählen

7. Button: Zeile anhängen

8. Button: Letzte Zeile löschen

9. Button: Spalte anhängen

10. Button: Letzte Spalte löschen

11. Selektionsbox :Zeilen

12. Selektionsbox: Spalten

13. Checkbox: Minimieren

14. Checkbox: Maximieren

15. Button: Solver starten


Die Buttons 1. bis 6. entsprechen der Funktionalität der Dialogoptionen in der Hauptmenüleiste. Die Buttons 7. bis 11. bzw. die beiden Selektionsboxen dienen zur Anpassung der Größe der Matrix des Zuordnungsproblems.
Für das Lösen eines Problems muss zuerst eine Matrix editiert werden. Nachdem ein Solver ausgewählt und die Art der der Optimierung (Minimierung / Maximierung -> Checkbox)ausgewählt wurde, kann der Solver über den Button 15 gestartet werden. Falls die Matrix nicht voll belegt ist, wird sie über einen Assistenten vervollständigt.

Die Lösung wird in einem neuen Fenster ausgegeben. Neben der Lösung werden hier auch die Lösungszeit und der verwendete Solver angegeben.

Anmerkung:

Für das Editieren der Matrix existiert ein Zwischenpuffer.
Falls die Matrix also versehentlich verkleinert bzw. Zeilen oder Spalten gelöscht wurden, werden diese zwischengespeichert und werden bei Vergrößerung der Matrix wieder in die entsprechenden Felder geschrieben.
