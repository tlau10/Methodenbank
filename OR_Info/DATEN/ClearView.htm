Transformer / ClearView

Motivation

Obwohl es viele Solver gibt, die präzise das Ergebnis und eine Vielfalt von weiteren Daten eines LP-Systems ausgeben, so ist LP-Solve jedoch vielfach die erste Wahl. Der Grund dafür ist einfach: das Ergebnis wird in einer übersichtlichen Form angezeigt. Andere Solver verbergen das Ergebnis gerade durch die vielen weiteren Informationen. Hinzu kommt, dass es unterschiedliche Formate gibt, die "von Hand" ausgewertet werden müssen. Mit diesem Projekt soll das Ausgabeformat der Solver standardisiert werden, wodurch eine einfachere Implementierung von Methodendatenbanktools ermöglicht wird. Dieses neue XML-Format wird von der Java-Konsolenanwendung Transformer aus den verscheidenen. Ausgabeformaten der Solver erstellt. Das Format wird von dem Tool ClearView in einer Form aufbereitet, die dem Benutzer eine einfache Auswertung der Solver-Daten ermöglicht.

Der Transformer

Der Transformer wandelt die verschiedenen Ergebnisdateien der derzeit an der FH eingesetzten Solver in XML-Dateien um. Dadurch wird eine einheitliche Struktur der Ergebnisdaten gewährleistet und erlaubt somit eine effiziente Entwicklung von Tools, die Solver benutzen. Das Tool wurde in Java entwickelt, um Plattformunabhängigkeit sicherzustellen. Außerdem bietet Java die Möglichkeit, reguläre Ausdrücke zu verwenden, die zum Parsen genutzt werden können (siehe unten).

Benutzerhandbuch von ClearView

Systemvoraussetzungen

ClearView läuft auf jedem Windows PC, der die folgenden Bedingungen erfüllt:
  • eine Installation vom Visual Studio .Net muss vorhanden sein, da mehrere DLLs für die Ausgabe benötigt werden
  • eine Version des Internet Explorers muss auf dem System installiert sein
  • das Verzeichnis c:\temp muss vorhanden sein und der Lesen und Schreiben muss darauf erlaubt sein
  • Java muss in der Version 1.4.2_03 oder höher installiert sein

Bedienung

Die Oberfläche von ClearView besteht aus drei Anzeigeseiten und vier Menüs. Beim Anzeigen von den XML Daten ist zu beachten, dass nur die Daten angezeigt werden können, welche der Transformer aus der Ausgabedatei des Solvers erstellen konnte.

Anzeigeseite Einfach

Auf dieser Seite erscheint in der Liste die Variablen und deren Ergebnisse. Im Feld "Zielfunktion" erscheint das Ergebnis der Zielfunktion. Im Feld "Solver" erscheint der Name des verwendeten Solvers und des Ausgabeformats Das Feld "Nachricht" zeigt eventuelle Nachrichten des Solvers an, wie z.B. wenn das Ergebnis nicht gültig ist.

Anzeigeseite System

Auf dieser Seite wird das Ausgangssystem dargestellt. Die Spalten sind: Name Hier steht der Name der Restriktion, und die Zielfunktion wird als "ZF" gekennzeichnet. "Min" und "Max" bezeichnen die Ober- und Untergrenzen der Variablen. Diese werden nur angezeigt, falls diese definiert wurden. Namen der Variablen Nach den Namen der Funktion folgen die Bezeichnungen der Variablen. In dieser Spalte stehen bei den Restriktionen den jeweiligen Faktor der Variable. Ist das Feld leer, so gilt in der Regel, dass der Wert implizit 0 dort ist. Sollte jedoch auch der Operator und der B Wert fehlen, so wurde die Restriktionen deklariert, aber nicht definiert. Dies kommt hauptsächlich bei Ergebnissen vom LP-Solve vor. Operator Bei den Restriktionen wird dort angezeigt, wie der Operator dort ist. Gültige Werte sind: <, <=, =, >, >=. Bei der Zielfunktion wird hier fest "->" angezeigt. B In dieser Spalte werden bei der Zielfunktion die Optimierungsrichtung, also "Min!" oder "Max!" angezeigt. Bei den Restriktionen erscheint dort der jeweilige B Wert.

Als Zeilen erscheinen

  • Die Zielfunktion
  • Eventuell vorhandene Restriktionen
  • Eventuell vorhandene Unter- und Obergrenzen

Anzeigeseite Details

Auf dieser Seite werden alle relevanten Daten dargestellt. Die ersten Spalten sind wie bei der Anzeigeseite System aufgebaut, mit dem Unterschied, dass bei der Zielfunktion in der Spalte Operator nun die Optimierungsrichtung angezeigt wird und in der B Spalte der Zielfunktionswert. Dazu kommen noch weiter Spalten: Dualwert Hier werden die Werte der Dualwerten zu der jeweiligen Restriktion angezeigt. Slack Sollte die Restriktion definiert worden sein, so wird hier der Wert des Slacks angezeigt. Sollte der Wert negativ oder Operator der Restriktion =, > oder >= sein, so ist der Wert immer 0. Außerdem wird eine weitere Zeile hinzugefügt, welche die Reduced Costs der Variablen ausgibt. Die letzte Zeile "Variablenwerte" nennt noch mal die Ergebnisse der jeweiligen Variablen.

Menü Datei

Öffnen
Dieser Befehl öffnet einen Dialog, in dem sie eine XML Datei mit LP Daten öffnen können. Sobald die Datei ausgewählt und bestätigt wurde, wird intern das Dokument mit dem vorhandenen Schema geprüft. Stimmt die Datei nicht mit den dort definierten Regeln überein, so wird eine Fehlermeldung ausgegeben. Die in der XML Datei vorhandenen Daten werden dann in den drei Anzeigeseiten dargestellt.

Speichern
Hier kann eine vorhandene XML Datei unter einem anderen Namen gesichert werden, oder eine neu erstellte Datei gesichert werden. Wenn eine Datei angezeigt wird, kann der neue Name und der Speicherort im Dialog ausgewählt werden. Dann wird die Datei dorthin kopiert.

Schließen
Die aktuelle Datei wird geschlossen.

Export nach HTML
Dies hat die selbe Wirkung wie das gleichnamige Menü.

Beenden
Das Programm wird beendet.

Einstellungen...
In dem sich öffnenden Dialog können die Pfade zu der benötigten Java Anwendung und zum Transformer angesehen und geändert werden. Mit der Schaltfläche "OK" werden die Einstellungen übernommen und der Dialog geschlossen, mit "Abbrechen" wird nur der Dialog beeendet.

Menü Bearbeiten
Dieses Menü hat einen einzigen Punkt: "System aus Zwischenablage kopieren"
Das Erstellen eines Systems über die Zwischenablage erfolgt so:

  1. Eine Solver Ausgabedatei wird in einem beliebigen Editor geöffnet.
  2. Der gesamte Inhalt wird ausgewählt. Modernere Editoren erlauben hier den Tastaturkürzel "STRG+A".
  3. Der ausgewählte Text wird kopiert. Normalerweise reicht hier der Tastaturkürzel "STRG+C"
  4. ClearView wird aufgerufen und der Menüpunkt ausgewählt. Alternativ kann der Tastaturkürzel "STRG+V" aufgerufen werden.

Nun wird im Verzeichnis c:\temp die Datei clearview.xml erstellt, die alle Daten aus dem Zwischenspeicher enthält. Das Ergebnis sollte jedoch abgespeichert werden, da diese Datei bei jeder Operation neu angelegt wird. Sollte es zu einem Fehler kommen, so erscheint eine Fehlermeldung.

Fehler
Es gibt Umstände, in denen ClearView nicht das macht, was zu erwarten wäre. Folgende Fehlerfälle wurden beobachtet:

  • Beim Einlesen aus der Zwischenablage erscheint die Fehlermeldung: "Temporäre Datei kann nicht gefunden werden".
    In diesem Fall war der Inhalt in der Zwischenablage kein lineares Programm, welches der transformer verarbeiten konnte. Dadurch erzeugte dieser nicht die Datei Clearview.xml, die nun aufgerufen wird.
  • Beim Anzeigen der Hilfe erscheint eine Fehlermeldung: Die Datei Hilfe.htm" muss sich im Verzeichnis mit der Clearview.exe befinden. Außerdem muss der Internet Explorer auf dem Rechner zur Verfügung stehen.
  • Beim Erstellen der HTML Statistik erscheint eine Fehlermeldung: Die Datei genhtml.xslt muss sich im Verzeichnis mit der Clearview.exe befinden
  • Beim Öffnen einer XML Datei erscheint eine Fehlermeldung: Beim Öffnen einer vorhandenen XML Datei wird diese mit dem Schema aus LP-Schema.xsd verglichen, die sich Verzeichnis mit der Clearview.exe befinden muss.
    Befindet sich in der Fehlermeldung der Hinweis "Validationsfehler", dann wird danach erklärt, warum diese XML Datei nicht gültig ist. Öffnen Sie eine andere Datei oder korrigieren Sie den Fehler in der Datei.
  • Bei der Ausgabe sind einige Felder leer: Die Qualität der Ausgabe ist nur so gut wie die Daten, die in der XML Datei stehen. Diese Daten wiederum stammen von verschiedenen Solvern, welche unterschiedlich viele Daten zur Verfügung stehen. Leere Felder können in der Regel als "0" interpretiert werden. Da es jedoch Ausnahmefälle gibt, wird auf eine Vorbelegung mit "0" verzichtet, um keine Fehler anzuzeigen.
