Extended Application

Version 2.0

Extended Application ist ein SW-Produkt zur Lösung linearer Programmierungsaufgaben.

Ein Problem kann in einem der folgenden Formate formuliert werden:

  • Equation Style
    hierbei versteht man die Formulierung eines Modells im Gleichungsstil nach syntaktischen Vorgaben
  • MPS Format
    Industriestandardformat der linearen Programmierung
  • Spreadsheet Style
    diese Art der Problemformulierung unterstützt Lotus-Dateien und Kompatible

Die einfachste Art XA einzusetzen erfolgt mit dem Equation Style (LP-Format). Im folgenden wird darauf eingegangen.

XA arbeitet im Batch-Modus, d.h. Daten werden dem System beim Aufruf mittels Parameter übergegeben. Der Equation Style erwartet dabei zwei Dateien. Das CLP-File (Command Line Parameter File), sowie das LP-File (Line Parameter-File). Beide Dateien können mit einem beliebigen Editor bearbeitet werden. Statt des CLP-Files können auch Startparameter verwendet werden.

Die Arbeitsweise von XA:
Das im Vorfeld formulierte Problem wird aus dem LP-File sequentiell ausgelesen. Die Berechnung wird durch die Parameter des CLP-Files gesteuert. Die Ausgabe der Berechnung erfolgt auf den Bildschirm, Drucker oder in eine im CLP-File angegebene Datei.

Die Installationsversion enthält ein Standard-CLP-File (XA.CLP), auf das XA zurückgreift, sofern kein anderes angegeben wird. Auf den Rechnern der FH sind i.d.R. die Files MUSTER.LP und MUSTER.CLP zu finden. Diese können kopiert und als Vorlage verwendet werden.

Vorgehensweise: 

  1. Kopieren der MUSTER-Files
     z.B.:   MUSTER.CLP -> PRODPLAN.CLP
             MUSTER.LP  -> PRODPLAN.LP
       bzw.  MUSTER.MPS -> PRODPLAN.MPS

  2. Problemspezifische Bearbeitung des CLP-Files. 
     Achten Sie hier auf den Eintrag des Ausgabe-
     mediums.

  3. Problemspezifische Bearbeitung des LP/MPS-Files, 
     also Formulierung des LP-Ansatzes.

  4. Start von XA:  XA  Dateiname.Ext
     Bsp.:          XA  PRODPLAN.CLP
