- O M P -

Dos-Programm, Tutorial-Version

OMP ist ein Programm zur Lösung linearer (LP) und gemischtganzzahliger Modelle (MIP). Verarbeitet werden OMP- und MPS-Dateien.
Leider zeigt das Programm nicht an, ob alternative Optimallösungen existieren. Bei MIP-Modellen wird je nach Baumabarbeitung die eine oder andere ganzzahlige Lösung gefunden.

In F111 ist das Handbuch zum Programm ausleihbar. Kurzbeschreibungen (Omp_kurz.doc, Omp_i2.doc) sind im Besf-Info-Verzeichnis auf dem Server zu finden.

MODULE

  • LP: revidierter Simplex-Algorithmus
  • Gemischtganzzahlige Programmierung (MIP, Branch & Bound Verfahren)
  • Modell Generator: zur Beschreibung umfassender Modelle

  • Autopilot: Bildschirmgenerator, Berichtsgenerator und Schnittstelle zu Lotus 1-2-3, sowie zu verschiedenen Datenbanken


PROGRAMMLAUF

  • AUFRUF
    omp4tutw.exe bzw. ompstart.bat im Verzeichnis \B&P (..siehe unten).
  • MODELL EINGEBEN (bei Neueingabe)
    'Edit Text' unter dem Menüpunkt 'File'. Speichern mit [ALT-S]. Zurück mit [ALT-X].
  • MODELL EINLESEN
    'Input' unter dem Menüpunkt 'Model'.
  • LÖSEN DES MODELLS
    'Solve Model' unter dem Menüpunkt 'Optimizer'. Je nach Version ist jetzt 'Execute' auszuführen. Zurück mit [ALT-X].

  • ERGEBNISSE EINSEHEN
    'Show Report' unter dem Menüpunkt 'Reports'. Zurück mit [ALT-X].


RECHNERKONFIGURATION (für Win9x/NT)

  • OMP kann unter Win9x nur im Verzeichnis ..\B&P installiert werden. Weil 'B&P' unter WinNT nicht darstellbar ist, wurde die Exe so 'gepatched', daß nun auch eine Installation im Verzeichnis ..\OMP möglich ist. Auf jeden Fall wird aber ein Ansi-Treiber benötigt.
  • In den Laborräumen wird der Ansi-Treiber nicht automatisch geladen (siehe config.sys), deshalb wird mittels OMPSTART.BAT der alternative Treiber ANSI.COM 'nachgeladen'.
  • Temp. Anpassung der Umgebungsvariable PATH in der OMPSTART.BAT

Aufruf von OMP ausschließlich via ompstart.bat.

Hinweis für NT-Nutzer:
Der Pfad \b&p ist unter NT nicht darstellbar. Eine für NT verträgliche Version ist im Verzeichnis OMP_NT zu finden. Die Exe wurde so geändert, dass nun der Pfad-Eintrag "omp" gesucht wird.
Bei Verwendung der zugehörigen Batch-Datei, wird automatisch das Verzeichnis c:\temp\omp angelegt.

RECHNERKONFIGURATION (für Dos/Win 3.x)

  • Hauptspeicherbedarf: kleiner 400 KB im unteren Bereich (Version 4).
  • OMP kann nur im Verzeichnis [root]\B&P installiert werden.
  • Die Umgebungsvariable PATH muss dieses Verzeichnis enthalten.
  • Der Bildschirmtreiber ANSI.SYS wird benötigt (config.sys).
