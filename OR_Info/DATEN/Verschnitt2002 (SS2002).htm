Verschnittplanung 2002

Im SS 2002 wurde das Methodenbankprogramm Verschnittplanung auf C# portiert.

Hinweise:

  • Eine gute Einführung in die Problematik bietet die Online-Hilfe des alten VB-Programms.
  • Grundsätzlich werden keine unteropitimalen Varianten erstellt, sondern nur bereits voroptimierte. Dies hält die Anzahl der möglichen Varianten klein. Deshalb kann der Verschnitt einer Variante nie "größer" als das kleinste Endprodukt sein. Dies kann wiederum zur Folge haben, dass zuviele Endprodukte gefertigt werden.
  • Die Portierung bietet mit ihrem objektorientierten Ansatz nun eine weitaus bessere Basis für Programmergänzungen u. Weiterentwicklungen.

Probleme bei der Nutzung unter NT 4.0:
Zum Aufruf des Solvers erstellt das Programm eine Batch-Datei, welche im Arbeitsverzeichnis c:\temp abgelegt wird. Leider wartet das Programm nicht auf das Ausführungsende nach dem Start der Batch-Datei.
Abhilfe: Berechnung durchführen (was ergebnislos bleibt, bzw. ein Ergebnis aus einem vorherigen Lauf einliest), die Batch-Datei "von Hand" starten und anschließend im Programm nochmals einen Berechnungslauf anstoßen.
