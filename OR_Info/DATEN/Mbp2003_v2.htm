Die Methode

Die Maschinenbelegungsplanung wird dann eingesetzt, wenn eine festgelegte Anzahl von Aufträgen mit entsprechenden Produktionszeiten in vorgegebenen Perioden (oder Alternativperioden, hier noch nicht implementiert) abgearbeitet werden soll. Den einzelnen Perioden werden Kapazitäten in Form von Maschinen unterschiedlicher Typen zugeordnet.
Die Aufgabe besteht nun darin, die Aufträge so auf die Perioden zu verteilen, dass die Auslastung der Maschinen über die Perioden (gleichmäßig) minimal ist. Weitere Optimierungsfälle sind denkbar, z.B. die möglichst frühe Abarbeitung der Aufträge, evtl. unter Einhaltung bestimmter Bedingungen, wie dem Freihalten festgelegter Kapazitäten usw.
Es lassen sich hier also noch viel Ideen in Arbeit umsetzen.

 

Im Mbp2003 wurde in Version 1 das Standard LP Modell zur Maschinenbelegungsplanung umgesetzt. Es gibt verschiedene Aufträge mit einer festgelegten Bearbeitungszeit und mögliche Perioden, in denen diese Aufträge gefertigt werden können. Für jede Periode steht eine bestimmte Anzahl an Maschinen zur Verfügung. Die Aufgabe bestand nun darin, die Aufträge derart auf Perioden und Maschinen zu verteilen, dass die Auslastung der Maschinen minimal wird.

 

In der Version 2 wurde das LP Modell dahingehend geändert, dass nun mehre Maschinentypen pro Periode berücksichtigt werden können.

 

Beispiel

Auftrag 1 hat eine Bearbeitungsdauer von 80 min., kann nur auf Maschinentyp 1 gefertigt werden und muss in den Perioden 1 und 2 abgearbeitet werden.

Auftrag 2 hat eine Bearbeitungsdauer von 100 min., kann ebenfalls nur auf Maschinentyp 1 gefertigt werden und muss in den Perioden 2, 3 und 4 abgearbeitet werden.

Auftrag 3 hat eine Bearbeitungsdauer von 50 min., kann auf Maschinentyp 1 und 2 gefertigt werden und muss in den Perioden  3 und 4 abgearbeitet werden.

  In Periode 1 steht eine Maschine des Typs 1 zur Verfügung, in Periode 2 stehen zwei Maschinen des Typs 1 zur Verfügung, in Periode 3 und 4 stehen jeweils eine Maschine der Typen 1 und 2 zur Verfügung.

Diese Aufträge werden unter Berücksichtigung der vorhandenen Kapazität in den einzelnen Perioden so eingeplant, dass die Auslastung der Maschinen minimal ist.

 

LP Ansatz:

  •   Minimierung der Zielfunktion:

            min: Y;

  •   Festlegung der Variablen

             1xA1P1T1 entspricht Auftrag 1,  Periode 1 und Maschinentyp 1.

  • Restriktionen

R1: 1xA1P1T1 + 1xA1P2T1                                                              = 80;

R2: 1xA2P2T1 + 1xA2P3T1 + 1xA2P4T1                                        = 100;

R3: 1xA3P3T1 + 1xA3P4T1 + 1xA3P3T2 + 1xA3P4T2                 = 50;

 

Da in Periode 3 und 4 auf beiden Maschinen gefertigt werden kann, muss eine Schaltervariablen für den Maschinentyp eingeführt werden.

  R4: 1xA3P3T1 + 1xA3P4T1 + 10000 - 10000 * MS3                    >= 50;

R5: 1xA3P3T2 + 1xA3P4T2 + 10000 * MS3                                   >= 50;

  • Periodenorientierte Glättung für Maschinentyp 1:

R6: 1xA1P1T1                                                                              <=  1Y;

R7: 1xA1P2T1 + 1xA2P2T1                                                         <=  2Y;

R8: 1xA2P3T1 + 1xA3P3T1                                                         <=  1Y;

R9: 1xA2P4T1 + 1xA3P4T1                                                         <=  1Y;

  • Periodenorientierte Glättung für Maschinentyp 2:

R10: 1xA3P3T2  <= 1Y;

R11: 1xA3P4T2  <= 1Y;

  • Restriktionen im Zusammenhang mit der maximalen Periodenlänge:

R12: 1xA1P1T1                                                               <= 60;

R13: 1xA1P2T1 + 1xA2P2T1                                          <= 120;

R14: 1xA2P3T1 + 1xA3P3T1                                          <= 60;

R15: 1xA2P4T1 + 1xA3P4T1                                          <= 60;

R16: 1xA3P3T2                                                               <= 60;

R17: 1xA3P4T2                                                               <= 60;

 

 

Das Programm

Das Programm MBP 2003 entstand im WS 2002/03 als Neuimplementierung in der Programmiersprache C#.
Im SS 2003 wurde das Programm für die Verwendung von mehreren Maschinentypen erweitert.
