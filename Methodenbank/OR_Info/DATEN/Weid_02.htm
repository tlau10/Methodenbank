Features of LP-optimizer

  • Implementation in Borland Pascal 7.0 (without Assembler) and Borland Delphi
    • The following Compiler-options should be set:
    • Word align
    • 286-Instructions
    • Numeric-Coprozessor active
  • Complete source code available
  • A single UNIT (go_lp) necessary to include the optimizer in own DOS or Win95 or Win-NT programs
  • Revised Simplex procedure
  • Mixed integer
  • sucessfully tested on the netlib collection of LP-problems
    All examples which fitted in to memory could be solved sucessfully
    Also most of the models from miplib library of mip-problems could be solved as well.
  • High numerical stability
  • Partial pricing
  • Sparce matrix storage
  • MPSX format read and write provided
  • Rowwise format read and write provided
  • Graphic display of matrix, different views dependent on matrix size
  • Basis save and restore
  • Crash Basis
  • Anti cycling procedure
  • epsilon perturbation on RHS
  • Output of demystified solution
  • Browsing through the matrix (Viewing of Rows and columns)
  • Many parameters, which can speed up optimisation (problem dependent)
  • View infeasible vectors
  • Context sensitive online help with hyperlinks (olny DOS-Version, DELPHI not yet ready)
  • Inversion routine based on Markoviz algorithm and preassigned pivot selection procedure
  • Maximal problem size:
  • DOS version:
    Rows, columns depending on free memory, can be specified in source code in unit lp_varia.pas
  • DPMI Version:
    Rows=6500 (64 k div 10 bytes, 8 for coefficient, 2 for row number)
    Rows+columns < 16384 (64k div 4)

    DPMI means: Dos Protected Mode Interface:
    This version runs on the following platforms:
  • DOS 3.3 and higher
  • DOS-Box of Windows 3.1 and higher
  • DOS-Box of Windows 95
  • DOS-Box of Windows NT
  • DOS-Box of OS/2
  • OS/2: In principle No limits
    The EXE-Version for download currently about 32000 ROWS and 32000 Columns
  • WINDOWS 95 or NT: In principle No limits
    The EXE-Version for download currently about 32000 ROWS and 130000 Columns
    Problem size is only limited by available memory and time
The OS/2 version was compiled with Virtual Pascal.
The WINDOWS-95 (NT) version was compiled with BORLAND DELPHI Version 2.
